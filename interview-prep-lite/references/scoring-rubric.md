# 评分维度与标准

本文件供 Agent 内部评分参考，不展示给用户。

---

## 简历六维度评分

### 维度与权重

| 维度 | 英文名 | 权重 | 评审重点 |
|---|---|---|---|
| 项目深度 | project_depth | 30% | 量化数据、技术选型理由、个人贡献、难点解决 |
| 技术匹配 | tech_match | 25% | 技术栈与目标岗位匹配度、技能描述有层次（精通/熟悉/了解） |
| 表达能力 | expression | 15% | 动词开头、STAR结构、无歧义、不冗长 |
| 结构合理 | structure | 15% | 模块完整、排版逻辑、信息密度、教育经历位置合理 |
| 量化程度 | quantification | 10% | 性能指标、用户量、优化幅度、业务结果 |
| 真实可信 | authenticity | 5% | 表述真实、深度与经验匹配、无明显夸大 |

### 评分标准（每个维度0-100）

- **80-100**：表现优秀，该维度几乎无问题，细节充实
- **60-79**：基本达标，有亮点但也有明显不足
- **40-59**：差距较大，该维度有较多问题需要改进
- **0-39**：不达标，该维度严重缺失或质量差

### 综合分计算

```
综合分 = Σ(维度得分 × 权重) / Σ(已评分维度权重)
```

缺维度时按已有权重重归一化，不按0分算。

### 评级分档

| 分数区间 | 评级 |
|---|---|
| 75-100 | 达标 |
| 60-74 | 基本达标 |
| 45-59 | 差距较大 |
| 0-44 | 不达标 |

### 问题优先级定义

- **高**：严重影响简历通过率，必须改（如无项目量化、技术栈与岗位不匹配、项目描述假大空）
- **中**：影响简历质量但不是致命问题，建议改（如技能未分级、排版问题、措辞不够精准）
- **低**：锦上添花的优化项，可改可不改（如个别措辞、格式微调）

## 面试五维度评分

### 维度与权重

| 维度 | 英文名 | 权重 | 评审重点 |
|---|---|---|---|
| 技术深度 | tech_depth | 35% | 知识掌握扎实度、概念理解准确性、原理解释深度、技术细节 |
| 项目经验 | project_experience | 25% | 个人贡献清晰度、选型理由、难点解决过程、量化结果 |
| 表达逻辑 | expression_logic | 20% | 条理性、STAR结构、语言清晰、回答切题不跑题 |
| 抗压反应 | stress_response | 10% | 不会的问题如何应对、思路调整、不慌张不冷场 |
| 整体印象 | overall_impression | 10% | 积极性、学习态度、岗位准备度、沟通氛围 |

### 回答质量分档（每道题）

| 档位 | 判断标准 |
|---|---|
| 好（excellent） | 覆盖80%以上要点，有具体技术细节/量化数据/原理解释，深度足够 |
| 一般（adequate） | 方向正确但缺乏深度，表述较笼统，覆盖50-79%要点 |
| 差（weak） | 方向偏差、过于表面或明显不理解，覆盖30-49%要点 |
| 不会（no_answer） | 明确说不知道、回答为空或内容无关，覆盖<30% |

### 综合分计算

同简历加权计算方式。

### 评级分档

| 分数区间 | 评级 |
|---|---|
| 90-100 | 远超初级岗位期望 |
| 75-89 | 达到初级岗位要求 |
| 60-74 | 基本达标，有明显提升空间 |
| 45-59 | 差距较大，需系统补强 |
| 0-44 | 尚未达到申请门槛 |

## 就绪度概率计算

复盘报告中的就绪度评估使用以下公式：

### 能投概率（简历就绪度）

```
能投概率 = min(100, 简历综合分) - 高优先级问题数 × 15
```
- 概率下限为0
- 高优先级问题数指简历中标记为"高"优先级的问题数量
- 每个高优先级问题拉低15分就绪度

### 能面概率（面试就绪度）

```
能面概率 = min(100, 面试综合分) - max(0, 50 - 最低维度分) × 2
```
- 概率下限为0
- 最低维度分指五维度中得分最低的那个维度的分数
- 如果最低维度分低于50分，每低1分拉低2分

### 能讲概率（项目表达就绪度）

```
能讲概率 = 计划完整度 × 100
```
- 计划完整度取决于7天/14天/30天三档计划是否都有具体内容
- 三档全有 = 100%，有两档 = 67%，只有一档 = 33%，都没有 = 0%
- （注意：能讲概率在首次面试中默认按三档计划都已生成算100%，因为报告里已经包含了完整计划）

### 等级划分

| 概率区间 | 等级 |
|---|---|
| ≥70% | 高（可以投/可以面/可以讲） |
| 40-69% | 中（有一定基础，建议再针对性准备） |
| <40% | 低（不建议，需要系统补强） |

### 阻塞项输出

当某档概率<70%时，必须输出结构化阻塞项：
- **描述**：具体是什么问题（引用具体的简历问题或面试薄弱维度，不能泛泛而谈）
- **差距**：量化差距（如"当前50分，距离60分及格线差10分"）
- **行动**：具体可执行的改进动作（不能说"加强学习"这种空话）

## 评分实操指引

### 简历评分——各维度打分参考示例

#### 项目深度（权重 30%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 每个项目有3+量化指标、技术选型有理由、难点有解决过程、个人贡献明确（"独立负责XX"而非"参与XX"） |
| 60-79 | 有量化数据但不够（1-2个）、选型理由只提名词不解释、个人贡献模糊（"参与"为主） |
| 40-59 | 无量化数据、纯描述性文字、看不出个人做了什么 |
| 0-39 | 项目描述假大空、复制粘贴感、与岗位无关 |

#### 技术匹配（权重 25%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 核心技术栈与岗位高度匹配、技能分三级（精通/熟悉/了解）、有对应的实践项目佐证 |
| 60-79 | 匹配度60-80%、技能未分级或分级不准、部分技能无项目佐证 |
| 40-59 | 匹配度<60%、技能全用"会/了解"无层次、核心技能缺失 |
| 0-39 | 技术栈与岗位几乎无关 |

#### 表达能力（权重 15%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 动词开头、STAR结构完整、无歧义、信息密度高不冗长、术语使用准确 |
| 60-79 | 基本清晰但部分描述冗长或模糊、个别术语使用不当 |
| 40-59 | 大段描述无重点、被动语态多、逻辑跳跃 |
| 0-39 | 语法混乱、无法理解意图、严重冗长 |

#### 结构合理（权重 15%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 模块完整（基本信息/技能/教育/工作/项目/评价）、排版逻辑清晰、教育经历位置合理、信息密度均匀 |
| 60-79 | 模块基本完整但排版有小问题、某个模块信息过多或过少 |
| 40-59 | 缺关键模块（如无技能或无项目）、排版混乱、信息头重脚轻 |
| 0-39 | 结构严重缺失、无法快速定位关键信息 |

#### 量化程度（权重 10%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 3+个量化指标（性能/用户量/优化幅度/F1/延迟等）、数据具体可信 |
| 60-79 | 1-2个量化指标、部分数据模糊（如"大幅提升"而非具体百分比） |
| 40-59 | 仅1个或不具体的量化、大部分用"显著""大幅"等模糊词 |
| 0-39 | 完全无量化数据 |

#### 真实可信（权重 5%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 深度与经验年限匹配、无明显夸大、表述谦逊有据、自我评价客观 |
| 60-79 | 个别表述略有夸大但整体可信、自我评价基本客观 |
| 40-59 | 有明显矛盾（如经验年限与经历不符）、自我评价有减分项（如"理论能力差"） |
| 0-39 | 严重夸大、多处矛盾、明显包装痕迹 |

### 面试评分——各维度打分参考示例

#### 技术深度（权重 35%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 能解释原理而非仅用法、能对比替代方案、能举边界条件和常见坑 |
| 60-79 | 知道用法和基本原理，但追问时缺乏深度或记不清细节 |
| 40-59 | 只知名词不能展开、概念混淆（如把RAG等同于关键词搜索） |
| 0-39 | 核心概念不了解、答非所问 |

#### 项目经验（权重 25%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 能清晰说清个人贡献、选型理由充分、难点有具体解决过程、量化结果能自洽 |
| 60-79 | 能描述项目但细节遗忘频繁、选型理由不够充分 |
| 40-59 | 只能复述简历文字、追问就卡住 |
| 0-39 | 项目细节完全答不上、与简历描述不符 |

#### 表达逻辑（权重 20%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 回答有结构（先总后分/STAR）、语言清晰不跑题、复杂问题能拆解 |
| 60-79 | 基本连贯但偶尔跑题或遗漏、复杂问题拆解不够 |
| 40-59 | 回答碎片化、经常跑题、逻辑跳跃 |
| 0-39 | 语无伦次、答非所问、长时间沉默 |

#### 抗压反应（权重 10%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 不会的问题坦诚说不知道、尝试给出思路、不慌张不冷场 |
| 60-79 | 不会时稍显紧张但能坦诚、基本不编答案 |
| 40-59 | 不会时明显慌张、长时间沉默、编造不确定的答案 |
| 0-39 | 完全僵住、编造明显错误答案、情绪崩溃 |

#### 整体印象（权重 10%）

| 得分区间 | 典型表现 |
|----------|----------|
| 80-100 | 积极主动、学习态度好、对岗位有准备、沟通氛围好 |
| 60-79 | 态度正常但不突出、准备度一般 |
| 40-59 | 态度消极或敷衍、明显未做岗位调研 |
| 0-39 | 态度问题严重、对岗位无兴趣 |

## 岗位权重微调

针对不同岗位方向，可微调面试五维度权重：

| 岗位方向 | 技术深度 | 项目经验 | 表达逻辑 | 抗压反应 | 整体印象 |
|----------|:--------:|:--------:|:--------:|:--------:|:--------:|
| Agent 开发 | 35% | 25% | 20% | 10% | 10% |
| LLM 应用 | 30% | 30% | 20% | 10% | 10% |
| 算法/NLP | 40% | 20% | 20% | 10% | 10% |
| 通用后端 | 25% | 30% | 25% | 10% | 10% |

默认使用"Agent 开发"权重，其他岗位按上表调整。
